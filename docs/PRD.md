## 2. 产品概述

### 2.1 产品定位

一款基于桌面端的智能随机点名工具，通过摄像头实时捕捉画面，结合人脸检测与手势/标志牌识别，实现公平、有趣、智能的随机抽选功能。

### 2.2 目标用户

- 教师（课堂随机提问、点名）
- 培训讲师（互动选人）
- 团队活动组织者（抽奖、分组）

### 2.3 核心价值

- 自动化：自动调用摄像头并实时识别人脸
- 公平性：随机算法保证抽选公平
- 智能化：识别 `X` 禁止符跳过、`✓` 对勾符号优选
- 趣味性：动态圈选动画 + 放大展示

## 3. 功能需求

### 3.1 功能清单

| 编号 | 功能模块 | 优先级 | 描述 |
| --- | --- | --- | --- |
| F-01 | 摄像头自动启动 | P0 | 程序启动后自动调用默认摄像头并实时显示画面 |
| F-02 | 实时人脸检测 | P0 | 检测画面中所有人脸并标记位置 |
| F-03 | 随机圈选动画 | P0 | 用圆圈在各人脸间随机跳动 |
| F-04 | 定时停止（5秒） | P0 | 抽选开始后 5 秒自动停止并锁定一人 |
| F-05 | 人脸放大展示 | P0 | 放大显示被选中的人脸 |
| F-06 | 重新抽选 | P0 | 提供按钮重新发起抽选 |
| F-07 | 退出程序 | P0 | 安全退出并释放摄像头 |
| F-08 | `X` 禁止符识别跳过 | P1 | 检测到持禁止符的人，从候选池剔除 |
| F-09 | `✓` 对勾符优先 | P1 | 检测到持对勾符的人，提高被选中权重 |
| F-10 | 无可选人提示 | P1 | 候选池为空时友好提示 |

### 3.2 功能详细说明

#### F-01 摄像头自动启动

- 程序入口为 `.exe` 文件
- 启动后自动枚举可用摄像头，默认选择第一个
- 异常处理：无摄像头或被占用时弹出提示
- 可选：设置页面允许切换摄像头设备

#### F-03 随机圈选动画

- 动画在所有“有效候选人脸”间随机跳动
- 圈选频率：初期快速（约 100ms/次），后期减速（缓动效果）
- 圆圈样式：醒目颜色（如黄色描边），跟随人脸框尺寸

#### F-04 / F-05 停止与放大

- 5 秒倒计时（可在配置中调整时长）
- 停止时圆圈停留在最终人脸上
- 截取该人脸区域，平滑放大动画展示于屏幕中央
- 展示放大画面同时显示“恭喜被选中”等提示

#### F-08 `X` 禁止符识别跳过

- 识别画面中的禁止/叉号标志牌
- 标志与人脸进行空间关联（标志位于人脸下方或附近）
- 关联人脸标记为“跳过”，从候选池移除，UI 显示灰色叉标记

#### F-09 `✓` 对勾符优先

- 识别对勾标志牌并关联人脸
- 该人脸权重提升（如普通权重 1，优选权重 3）
- UI 显示绿色对勾标记

## 4. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 实时画面 ≥ 20 FPS；人脸检测延迟 < 200ms |
| 兼容性 | Windows 10/11（64 位），支持打包为单文件 exe |
| 易用性 | 主界面操作不超过 3 个按钮，零学习成本 |
| 稳定性 | 连续运行 30 分钟无崩溃，正确释放摄像头资源 |
| 隐私 | 所有处理本地完成，不上传任何图像数据 |

## 5. 交互流程

```text
启动程序
   │
   ▼
自动调用摄像头 ──失败──► 错误提示 ──► 退出/重试
   │成功
   ▼
实时显示画面 + 人脸检测 + 标志识别
   │
   ▼
点击「开始抽选」
   │
   ▼
过滤候选池（剔除 X，加权 ✓）
   │
   ├─候选为空──► 提示「无可抽选对象」──► 返回
   │候选非空
   ▼
随机圈选动画（圆圈跳动，逐渐减速）
   │
   ▼
5 秒后停止 ──► 锁定人脸 ──► 放大展示
   │
   ▼
显示操作按钮：
   ├─「重新抽选」──► 回到过滤候选池
   └─「退出」────► 释放资源并关闭
```
